# 16位通用移位寄存器设计

## 设计内容

设计一个16位移位寄存器，要求移位方式和移位位数可变。

## 设计工具

ModelSim 用于对设计进行仿真验证。

## 设计步骤

### 1. 16位循环右移器

16位循环右移的实现采用16片4bit2路复用器，分为四级串联，每级分别移位**8**,**4**,**2**,**1**或**0**位，最终可实现0到15位的循环移位功能。移位功能通过输入端信号的不同接法实现，电路原理图如[图1](#pic_1)所示，封装模块如[图2](#pic_2)所示。

### 2. 通用移位寄存器

对需要设计的移位电路进行需求分析：

1. 右移：
   1. 循环右移：原电路本身能实现循环右移功能；
   2. 逻辑右移：低位移动，高位补**1**。移动位数为**k**时，可将**1**赋给输入**din**的低**k**位，可以在循环移位器的基础上实现逻辑右移功能；
   3. 算术右移：高位符号位不变，低位移动。同逻辑右移，将**data_in[15]**赋给输入**din**的低**k**位；
2. 左移：
   1. 循环左移：左移**k**位相当于右移**16-k**位，因此循环左移可以通过原电路右移实现；
   2. 逻辑左移：高位移动，最低为补**0**。在循环左移的基础上，将**0**赋给**din**的高**k**位，则左移后高**k**位变为低**k**位，实现逻辑左移功能；
   3. 算术左移：同逻辑左移。

总结：设置标志**mode[2:0]**表示移位种类和移位方向，标志信号功能定义如下表：

| mode[2] | mode[1] | mode[0] | 功能           |
| ------- | ------- | ------- | -------------- |
| 1       | x       | 0       | 循环左移       |
| 1       | x       | 1       | 循环右移       |
| 0       | x       | 0       | 逻辑、算数左移 |
| 0       | 0       | 1       | 逻辑右移       |
| 0       | 1       | 1       | 算术右移       |

#### 2.3 实现不同移位方式输入数据的选择

**mode[0]=0**时左移，左移**k**位时对高**k**位进行数据选择输入。**mode[2:1]=10**时循环移位，高**k**位保持不变；**mode[2:1]=00**时逻辑移位，高**k**位赋为**0**；**mode[2:1]=01**时算术移位，高**k**位赋为**0**；

**mode[0]=1**时右移，右移**k**位时对低**k**位进行数据选择输入。**mode[2:1]=10**时循环移位，低**k**位保持不变；**mode[2:1]=00**时逻辑移位，低**k**位赋为**0**；**mode[2:1]=01**时算术移位，高**k**位赋为**data[15]**；

**mode[2:1]=11**时定义为循环移位。

| mode[2] | mode[1] | mode[0] | 功能           | 对输入数据操作    |
| ------- | ------- | ------- | -------------- | ----------------- |
| 1       | x       | 0       | 循环左移       | 无                |
| 1       | x       | 1       | 循环右移       | 无                |
| 0       | x       | 0       | 逻辑、算数左移 | 高k位赋为0        |
| 0       | 0       | 1       | 逻辑右移       | 低k位赋位0        |
| 0       | 1       | 1       | 算术右移       | 低k位赋位data[15] |



对输入数据的操作采用逻辑位运算完成，此时需要生成三组用于进行逻辑运算的16位数据，

1. **d_lsh**：高k位为0、其余位为1的16位数据，将其与原始输入数据**data_in**进行按位与运算，得到逻辑、算术左移输入数据；
2. **d_rsh**：低k位为0、其余位为1的16位数据，将其与原始输入数据**data_in**进行按位与运算，得到逻辑右移输入数据；
3. **d_sar**：低k位为1、其余位为0的16位数据，将其与原始输入数据**data_in**进行一系列运算（下表给出具体公式），得到算术右移输入数据

| 名称  | 定义                          | 功能                           |
| ----- | ----------------------------- | ------------------------------ |
| d_lsh | 高k位为0、其余位为1的16位数据 | 用于得到逻辑、算术左移输入数据 |
| d_rsh | 低k位为0、其余位为1的16位数据 | 用于得到逻辑右移输入数据       |
| d_sar | 低k位为1、其余位为0的16位数据 | 用于得到算术左移输入数据       |

## 仿真结果

使用Verilog编写testbench进行仿真，代码如下，

```verilog
module testbench;

    reg clk, rstn, load;
    reg [15:0] data_in;
    reg [3:0] shift;
    reg [2:0] mode;
    wire [15:0] data_out;

    univ_shift_reg UUT(clk, rstn, load, data_in, shift, mode, data_out);

    initial begin
        clk = 0;
    end
    always begin
        #5  clk = ~clk;
    end

    initial begin
        #0
            rstn = 0;
        #10
            rstn = 1;
            load = 1;
            data_in = 16'b1100_0110_1001_0011;
        #10
            load = 0;
            mode = 3'b100;      //rol
            shift = 4'b0011;
        #90
            load = 1;
        #10
            load = 0;
            mode = 3'b101;      //ror
            shift = 4'b0010;
        #60
            load = 1;
        #10
            load = 0;
            mode = 3'b000;      //lsh
            shift = 4'b011;
        #40
            load = 1;
        #10
            load = 0;
            mode = 3'b001;      //shr
            shift = 4'b100;
        #40
            load = 1;
        #10
            load = 0;
            mode = 3'b011;      //sar
            shift = 4'b010;
        #90 $finish();
    end

endmodule //testbench
```

## 设计总结

本设计采用四级4位2选1数据选择器，电路整体延迟较低，较大程度上降低了竞争冒险的时间。但本设计使用了较多数据选择器和译码器，门电路多，硬件资源消耗较大，是本设计需要改进之处。
